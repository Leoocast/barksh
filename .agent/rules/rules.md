# Reglas del Workspace

## Comunicación

1. **Respuestas concisas**
   - Priorizar listas y bloques de código sobre párrafos largos.

2. **Español casual**
   - Tuteo. Sin formalidades innecesarias.

3. **Opciones claras**
   - Cuando existan decisiones, presentar alternativas en tabla con pros y contras.

4. **Pregunta antes de actuar**
   - Si hay ambigüedad, preguntar: **“¿Cuál prefieres?”** en lugar de asumir.

5. **Confirma entendimiento**
   - Reformular brevemente lo entendido antes de implementar.

---

## Código

1. **Edits mínimos**
   - Modificar solo lo necesario.
   - No refactorizar sin solicitud explícita.

2. **Explicar el “por qué”**
   - Dar contexto breve de decisiones técnicas no obvias.

3. **Mostrar el uso**
   - Después de crear funciones o utilidades, incluir ejemplo de uso.

---

## Flujo de Trabajo

1. **Seguir el ritmo del usuario**
   - Si el usuario modifica archivos manualmente, observar y adaptarse.
